Prototype: Smart Pinning UX

Let the router decide when a thread should reroute.

This prototype treats smart pinning as a per-thread horizon. The classifier can suggest how many future user turns should stay pinned, while tool loops stay free and do not burn the reroute budget.

Classifier suggests `rerouteAfterUserTurns` User turns consume budget Tool loops stay pinned for free Admins can clamp or disable it
